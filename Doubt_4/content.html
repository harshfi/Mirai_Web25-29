Understanding Arrays in C++

A concise guide for beginners — declaration, initialization, traversal, pitfalls, and alternatives.

Estimated read: 6 min•Author: Harsh

What is an array?

An array is a contiguous block of memory that stores elements of the same type. Arrays are useful when you want to group fixed-size collections of values and access them by index.

Tip: In C++ array indices start at 0. The last element of an array with size n is at index n-1.

Declaring and initializing arrays

Static (C-style) arrays are declared with a type and size. Here are common patterns:

// declaration (uninitialized)
int a[5];

// declaration + initialization
int b[5] = {1, 2, 3, 4, 5};

// partial initialization (rest are zero)
int c[5] = {10, 20}; // c = {10,20,0,0,0}

// size inferred from initializer
int d[] = {7, 8, 9}; // size = 3

// character array / string literal (not std::string)
char s[] = "hi"; // size = 3 (h, i, '\0')

Be careful: a declaration like int a[-3]; is invalid — size must be a constant positive integer in standard C++ (variable-length arrays are not part of standard C++).

Accessing and traversing

Use the index operator []. Typical traversal uses a for loop.

for (int i = 0; i < 5; ++i) {
    std::cout << a[i] << '\n';
}

// C++ range-based for (since C++11)
for (int x : b) {
    std::cout << x << '\n';
}

Always ensure the index is within bounds. Accessing out-of-range indices causes undefined behavior.

Multidimensional arrays

// 2D array 3x4
int mat[3][4];

// initialize
int grid[2][3] = {{1,2,3},{4,5,6}};

// iterate
for (int i = 0; i < 2; ++i)
  for (int j = 0; j < 3; ++j)
    std::cout << grid[i][j] << ' ';

Multidimensional arrays are stored in row-major order in memory.

Passing arrays to functions

When you pass a built-in array to a function, it decays to a pointer to its first element — the function loses knowledge of the array size:

void printArr(int *arr, int n) {
  for (int i = 0; i < n; ++i) std::cout << arr[i] << '\n';
}

int main() {
  int a[3] = {10,20,30};
  printArr(a, 3); // pass pointer + size
}

If you want safer alternatives that preserve size information, see std::array and std::vector below.

Common pitfalls

  • Out-of-bounds access → undefined behavior (often crashes or silent bugs).
  • Uninitialized arrays contain garbage values when declared without initialization.
  • Beware of pointer decay when passing arrays to functions.
  • Using sizeof on a decayed pointer won't give array size. Use template helper or pass size explicitly.

Modern C++ alternatives

Prefer these in most cases:

// fixed-size container (wraps C-style array)
#include <array>
std::array<int,5> arr = {1,2,3,4,5};

// dynamic-size container
#include <vector>
std::vector<int> vec = {1,2,3};
vec.push_back(4);

// safer traversal
for (auto &x : vec) std::cout << x << '\n';

std::array keeps size information and behaves like a stack-allocated fixed-size array. std::vector is dynamic, manages memory, and is the go-to container for variable-length arrays.

Examples — small programs

// Example: sum of elements
int sum(int *a, int n) {
  int s = 0;
  for (int i = 0; i < n; ++i) s += a[i];
  return s;
}

// Example: reverse an array in-place
void reverseArr(int a[], int n) {
  for (int i = 0; i < n/2; ++i)
    std::swap(a[i], a[n-1-i]);
}

Quick reference

  • Indexing: a[i] (0 ≤ i < n)
  • Size known at compile-time: use std::array or built-in array
  • Dynamic size: use std::vector
  • Strings: prefer std::string over char[] for safer string handling
